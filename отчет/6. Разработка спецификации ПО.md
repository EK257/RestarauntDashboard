# Спецификация программного обеспечения

## 1. Архитектура системы

*   **Тип приложения:** Desktop-приложение с графическим интерфейсом
*   **Архитектурный паттерн:** Модель-Представление-Контроллер (MVC)
*   **Хранение данных:** Локальная реляционная база данных
*   **Сетевое взаимодействие:** Отсутствует (автономная работа)

## 2. Модули системы

### 2.1. Модуль панели управления (Dashboard)
*   **Назначение:** Центральный рабочий инструмент администратора для ежедневной работы
*   **Функциональность:**
    *   Отображение актуальной информации в реальном времени
    *   Быстрое бронирование для пришедших клиентов
    *   Управление статусами посещений
    *   Автоматическое обновление данных

### 2.2. Модуль управления бронированиями
*   **Назначение:** Полный цикл работы с бронированиями
*   **Функциональность:**
    *   Создание новых бронирований
    *   Редактирование существующих записей
    *   Отмена и удаление броней
    *   Поиск ближайшего доступного времени

### 2.3. Модуль управления столиками
*   **Назначение:** Учет и управление столиками ресторана
*   **Функциональность:**
    *   Добавление новых столиков
    *   Изменение параметров столиков
    *   Удаление столиков (с проверкой зависимостей)
    *   Управление статусами столиков

### 2.4. Модуль управления зонами
*   **Назначение:** Организация пространства ресторана
*   **Функциональность:**
    *   Создание новых зон
    *   Удаление ненужных зон

## 3. Интерфейсы системы

### 3.1. Интерфейс базы данных
*   **Структура таблиц:**
    *   Таблица `tables`
    *   Таблица `zones`
    *   Таблица `clients`
    *   Таблица `reservations`

### 3.2. Бизнес-интерфейсы
*   **Интерфейс управления бронированиями:**
    *   Проверка доступности столика на заданное время
    *   Автоматическая генерация времени окончания
    *   Проверка пересечений временных окон
*   **Интерфейс управления столиками:**
    *   Валидация параметров столика (вместимость, статус)
    *   Проверка возможности удаления столика
    *   Автоматическое обновление статусов
*   **Интерфейс аналитики:**
    *   Расчет статистики загрузки

## 4. Алгоритмы работы

### 4.1. Алгоритм бронирования столика
*   **Входные параметры:**
    *   Имя клиента
    *   Дата и время посещения
    *   Количество гостей
    *   Желаемая длительность
*   **Процесс:**
    1.  Валидация входных данных
    2.  Поиск подходящих столиков по вместимости
    3.  Проверка доступности на выбранное время
    4.  Расчет времени окончания посещения
    5.  Создание записи о клиенте (при необходимости)
    6.  Создание бронирования в базе данных
    7.  Обновление статуса столика
    8.  Возврат результата операции

### 4.2. Алгоритм автоматического обновления статусов
*   **Процесс:**
    1.  Получение текущей даты и времени
    2.  Для каждого столика:
        *   Проверка наличия активной брони в текущий момент
        *   Проверка наличия будущих бронирований
        *   Определение нового статуса столика
    3.  Обновление статуса в базе данных при изменении
    4.  Пересчет общей статистики загрузки

### 4.3. Алгоритм поиска ближайшего доступного времени
*   **Входные параметры:**
    *   Количество гостей
    *   Желаемая длительность посещения
*   **Процесс:**
    1.  Определение временных окон (с 10:00 до 23:00, шаг 15 минут)
    2.  Поиск начиная с текущей даты (максимум 30 дней вперед)
    3.  Для каждой даты и времени:
        *   Поиск доступных столиков по вместимости
        *   Проверка отсутствия пересечений с существующими бронями
    4.  Возврат первого найденного варианта

## 5. Ограничения системы

### 5.1. Функциональные ограничения
*   **Временные ограничения:**
    *   Минимальный интервал бронирования: 15 минут
    *   Максимальная длительность брони: 180 минут (3 часа)
    *   Время работы ресторана: 10:00 - 23:00
    *   Бронирование возможно только на будущие даты
    *   Автоматическое обновление статусов: каждые 5 минут
*   **Ограничения по данным:**
    *   Максимальная вместимость столика: 50 человек
    *   Минимальная вместимость: 1 человек
    *   Статусы столиков: "Свободен", "Занят", "Забронирован", "На ремонте"
    *   Статусы бронирований: "Подтверждено", "Активно", "Завершено", "Отменено", "Неявка"

### 5.2. Ограничения пользовательского интерфейса
*   **Навигация:**
    *   Только последовательный переход между разделами
    *   Отсутствие многопользовательского режима
    *   Невозможность одновременного редактирования одних данных
*   **Ввод данных:**
    *   Ограниченный набор временных слотов (15-минутные интервалы)
    *   Фиксированные значения длительности посещения
    *   Предопределенные варианты количества гостей

### 5.3. Технические ограничения
*   **Производительность:**
    *   Работа с базой данных до 1000 записей бронирований
    *   Одновременное отображение до 100 столиков
    *   Время отклика интерфейса: до 100 мс
    *   Время загрузки данных: до 2 секунд
*   **Хранение данных:**
    *   Локальное хранение (без облачной синхронизации)
    *   Отсутствие резервного копирования 
    *   Ограниченная история бронирований

### 5.4. Бизнес-ограничения
*   **Управление бронированиями:**
    *   Невозможность бронирования на уже занятое время
    *   Запрет удаления столиков с активными бронями
    *   Запрет удаления зон с привязанными столиками
    *   Столики "На ремонте" исключаются из статистики
*   **Рабочие процессы:**
    *   Только ручной ввод бронирований (без онлайн-бронирования)
    *   Отсутствие интеграции с платежными системами
    *   Нет модуля учета финансов
    *   Отсутствие печати чеков и отчетов

### 5.5. Ограничения безопасности
*   **Доступ:**
    *   Отсутствие системы аутентификации
    *   Нет разграничения прав доступа
    *   Все данные доступны любому пользователю системы
*   **Защита данных:**
    *   Локальное хранение без шифрования
    *   Отсутствие защиты от SQL-инъекций (кроме базовой)
    *   Нет журналирования действий пользователей

## 6. Требования к окружению

### 6.1. Минимальные требования
*   Операционная система: Windows 10 / macOS 10.14 / Linux с графической оболочкой
*   Процессор: 1 ГГц или выше
*   Оперативная память: 2 ГБ
*   Свободное место на диске: 100 МБ
*   Java Runtime Environment: версия 8 или выше

### 6.2. Рекомендуемые требования
*   Оперативная память: 4 ГБ
*   Процессор: 2 ядра, 2 ГГц
*   Свободное место на диске: 500 МБ
*   Монитор: разрешение 1920x1080 или выше
