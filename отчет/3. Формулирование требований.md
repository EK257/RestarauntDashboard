# Функциональные требования

## 1.1. Управление бронированиями

### 1. Просмотр бронирований:
- Отображение всех бронирований в табличном виде
- Фильтрация бронирований по выбранной дате
- Сброс фильтрации для показа всех бронирований

### 2. Создание бронирования:
- Ввод данных клиента (имя)
- Выбор даты бронирования (календарь)
- Выбор времени начала (выпадающий список с шагом 15 минут)
- Выбор длительности (15, 30, 45, 60, 75, 90, 105, 120, 135, 150, 165, 180 минут)
- Указание количества гостей (1, 2, 3, 4, 5, 6, 8, 10, 12)
- Автоматический подбор доступных столиков по заданным параметрам
- Проверка на пересечение с существующими бронированиями

### 3. Редактирование бронирований:
- Изменение статуса бронирования
- Удаление бронирования

### 4. Автоматические функции:
- Расчет времени окончания брони на основе длительности
- Автоматическое создание клиента при вводе имени

## 1.2. Управление столиками

### 1. Просмотр столиков:
- Табличное отображение всех столиков
- Показ статусов столиков (Свободен, Занят, Забронирован, На ремонте)

### 2. Редактирование столиков:
- Добавление нового столика с указанием вместимости, зоны и статуса
- Редактирование существующего столика
- Удаление столика (только если нет активных бронирований)

### 3. Автоматические функции:
- Автоматическое обновление статусов столиков каждые 5 минут
- Автоматическая нумерация столиков

## 1.3. Управление зонами

### 1. Просмотр зон:
- Отображение списка всех зон ресторана

### 2. Редактирование зон:
- Добавление новой зоны
- Удаление зоны (только если в ней нет столиков)

## 1.4. Панель управления (Dashboard)

### 1. Обзорная информация:
- Просмотр бронирований на текущий день
- Отображение всех столиков с их статусами
- Показатель загрузки зала в процентах

### 2. Быстрые действия:
- "Бронь сейчас" - бронирование для клиентов, которые пришли в ресторан
- "Бронь позже" - предварительное бронирование на будущую дату
- "Найти лучшее время" - автоматический поиск ближайшего доступного времени

### 3. Управление статусами:
- "Клиент прибыл" - изменение статуса брони на "Активно"
- "Завершить" - завершение текущей брони
- "Отменить" - отмена бронирования
- "Освободить столик" - принудительное освобождение столика

### 4. Интеллектуальные функции:
- Автоматический поиск ближайшего доступного времени
- Поиск лучшего времени и даты на ближайшие 30 дней

## 1.5. Главный экран

### 1. Навигация:
- Переход на панель управления
- Переход в раздел бронирований
- Переход в раздел столиков
- Переход в раздел зон

### 2. Статистика:
- Отображение текущей загрузки зала в процентах
- Цветовая индикация загрузки (зеленый/желтый/красный)

# Нефункциональные требования

## 1. Производительность

### Время отклика:
- Загрузка данных в таблицы: не более 2 секунд
- Отклик на действия пользователя: не более 100 мс

## 2. Надежность
- Автоматическая проверка пересечений бронирований
- Валидация вводимых данных
- Предотвращение удаления данных с активными связями

## 3. Удобство использования

### 1. Интерфейс:
- Единый стиль всех экранов
- Интуитивная навигация между разделами
- Четкие названия кнопок и полей
- Контекстные подсказки

### 2. Визуализация:
- Цветовая индикация уровня загрузки
- Четкое разделение информации на экранах

### 3. Эффективность работы:
- Минимальное количество кликов для частых операций
- Автоматическое заполнение полей, где возможно
- Диалоговые окна для подтверждения важных действий

## 4. Безопасность

### 1. Защита данных:
- Локальное хранение данных (без отправки в интернет)
- Валидация ввода для предотвращения SQL-инъекций
- Корректное закрытие соединений с БД

### 2. Целостность операций:
- Проверка доступности столиков перед бронированием
- Предотвращение двойных бронирований

## 5. Технические требования

### 1. Платформа:
- Java 8 или выше
- JavaFX для графического интерфейса
- SQLite для хранения данных
- Кроссплатформенность (Windows, macOS, Linux)

### 2. Ресурсы:
- Минимальные требования: 2 ГБ ОЗУ
- Дисковое пространство: 100 МБ
- Не требует постоянного подключения к интернету
