# Прецеденты

## 1. Быстрая бронь для пришедшего клиента
**Название:** Бронь сейчас  
**Актор:** Администратор  
**Цель:** Быстро посадить клиента, который пришел в ресторан без брони  

**Основной сценарий:**
1. На Панели управления нажимает "Бронь сейчас"
2. Вводит имя клиента
3. Указывает количество гостей
4. Система показывает доступные столики
5. Выбирает столик
6. Нажимает "Посадить сейчас"
7. Система создает бронь со статусом "Активно"

**Исключения:**
- Исключение 1: Нет свободных столиков – система предлагает подождать или выбрать другое время

## 2. Предварительное бронирование
**Название:** Бронь позже  
**Актор:** Администратор  
**Цель:** Забронировать столик на будущую дату для клиента  

**Основной сценарий:**
1. На Панели управления нажимает "Бронь позже"
2. Вводит имя клиента
3. Выбирает дату и время
4. Указывает количество гостей
5. Нажимает "Найти лучшее время" (опционально)
6. Система показывает доступные столики
7. Выбирает столик
8. Нажимает "Забронировать"
9. Система создает бронь со статусом "Подтверждено"

**Исключения:**
- Исключение 1: На выбранное время нет свободных столиков – система предлагает другую дату или время

## 3. Отметка прибытия клиента с бронированием
**Название:** Клиент прибыл  
**Актор:** Администратор  
**Цель:** Подтвердить прибытие клиента с существующей брони  

**Основной сценарий:**
1. На Панели управления находит бронирование клиента
2. Нажимает кнопку "Клиент прибыл"
3. Система показывает подтверждение
4. Администратор подтверждает
5. Система меняет статус брони на "Активно"
6. Статус столика меняется на "Занят"

**Исключения:**
- Исключение 1: Клиент пришел, но его столик еще занят – система предлагает выбрать другой столик

## 4. Завершение посещения клиента
**Название:** Завершить посещение  
**Актор:** Администратор  
**Цель:** Отметить уход клиента и освободить столик  

**Основной сценарий:**
1. На Панели управления находит активное бронирование (статус "Активно")
2. Нажимает кнопку "Завершить"
3. Система показывает подтверждение
4. Администратор подтверждает
5. Система меняет статус брони на "Завершено"
6. Статус столика меняется на "Свободен"

## 5. Отмена бронирования
**Название:** Отмена брони  
**Актор:** Администратор  
**Цель:** Отменить существующее бронирование по просьбе клиента  

**Основной сценарий:**
1. На Панели управления или в разделе Бронирования находит бронь
2. Нажимает кнопку "Отменить"
3. Система показывает подтверждение
4. Администратор подтверждает
5. Система меняет статус брони на "Отменено"
6. Столик освобождается

**Исключения:**
- Исключение 1: Клиент хочет не отменить, а перенести бронь – администратор редактирует время

## 6. Добавление нового бронирования через раздел
**Название:** Добавление брони  
**Актор:** Администратор  
**Цель:** Создать новое бронирование из раздела Бронирования  

**Основной сценарий:**
1. В разделе "Бронирования" нажимает "Добавить бронирование"
2. Заполняет все поля: клиент, дата, время, длительность, гости
3. Система проверяет доступность столиков
4. Показывает список доступных столиков
5. Выбирает столик
6. Нажимает "Добавить"
7. Система создает бронирование

**Исключения:**
- Исключение 1: Нет свободных столиков – система предлагает изменить параметры

## 7. Редактирование бронирования
**Название:** Изменение брони  
**Актор:** Администратор  
**Цель:** Изменить параметры существующей брони  

**Основной сценарий:**
1. В разделе "Бронирования" выбирает бронирование
2. Нажимает "Изменить"
3. Меняет нужные параметры (время, столик, количество гостей)
4. Система проверяет доступность
5. Сохраняет изменения

**Исключения:**
- Исключение 1: Новое время уже занято – система предлагает альтернативные варианты

## 8. Добавление нового столика
**Название:** Добавление столика  
**Актор:** Администратор ресторана  
**Цель:** Добавить новый столик в систему  

**Основной сценарий:**
1. В разделе "Столики" нажимает "Добавить столик"
2. Указывает вместимость (количество мест)
3. Выбирает зону из списка
4. Выбирает начальный статус
5. Нажимает "Добавить"
6. Система создает столик

**Исключения:**
- Исключение 1: Не выбрана зона – система требует выбрать зону перед сохранением

## 9. Редактирование столика
**Название:** Изменение столика  
**Актор:** Администратор  
**Цель:** Изменить параметры существующего столика  

**Основной сценарий:**
1. В разделе "Столики" выбирает столик
2. Нажимает "Изменить столик"
3. Меняет параметры (вместимость, зону, статус)
4. Нажимает "Сохранить"
5. Система обновляет данные столика

**Исключения:**
- Исключение 1: Попытка изменить столик с активной броней – система предупреждает о конфликте

## 10. Удаление столика
**Название:** Удаление столика  
**Актор:** Администратор  
**Цель:** Удалить столик из системы  

**Основной сценарий:**
1. В разделе "Столики" выбирает столик
2. Нажимает "Удалить столик"
3. Система проверяет наличие активных броней
4. Если броней нет – показывает подтверждение
5. Администратор подтверждает
6. Система удаляет столик

**Исключения:**
- Исключение 1: На столике есть активные брони – система блокирует удаление

## 11. Добавление новой зоны
**Название:** Добавление зоны  
**Актор:** Администратор  
**Цель:** Создать новую зону в ресторане  

**Основной сценарий:**
1. В разделе "Зоны" нажимает "Добавить зону"

3. Нажимает "Добавить"
4. Система создает зону

**Исключения:**
- Исключение 1: Зона с таким названием уже существует – система предлагает другое название

## 12. Удаление зоны
**Название:** Удаление зоны  
**Актор:** Администратор  
**Цель:** Удалить зону из системы  

**Основной сценарий:**

2. Нажимает "Удалить зону"
3. Система проверяет, есть ли столики в зоне
4. Если столиков нет – показывает подтверждение
5. Администратор подтверждает
6. Система удаляет зону

**Исключения:**
- Исключение 1: В зоне есть столики – система блокирует удаление

## 13. Поиск лучшего времени для брони
**Название:** Поиск лучшего времени  
**Актор:** Администратор  
**Цель:** Найти ближайшее доступное время для бронирования  

**Основной сценарий:**
1. При создании брони администратор указывает количество гостей
2. Нажимает кнопку "Найти лучшее время"
3. Система анализирует расписание на 30 дней вперед
4. Находит первый свободный слот с доступным столиком
5. Автоматически заполняет дату и время
6. Показывает доступные столики

**Исключения:**
- Исключение 1: Нет свободных слотов на 30 дней – система сообщает об этом
