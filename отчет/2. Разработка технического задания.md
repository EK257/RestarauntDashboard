# Разработка технического задания

## 1\. Название проекта

**RestaurantDashboard** – система управления бронированиями и столиками для ресторанов

## 2\. Цели разработки

* Автоматизация процесса бронирования столиков в ресторане
* Управление статусами столиков в реальном времени
* Анализ загрузки зала
* Упрощение работы администратора

## 3\. Описание системы

Desktop-приложение на Java с графическим интерфейсом JavaFX для управления бронированиями столиков в ресторане. Система хранит данные в локальной базе SQLite и работает без подключения к интернету.

## 4\. Целевая аудитория

Администраторы ресторанов, управляющие, владельцы малого ресторанного бизнеса.

## 5\. Общие требования

### 5.1. Функциональные требования:

**Управление бронированиями:**

* Просмотр всех бронирований в таблице
* Фильтрация бронирований по дате
* Добавление новых бронирований с выбором столика
* Удаление бронирования

**Управление столиками:**

* Просмотр всех столиков с их статусами
* Добавление/удаление/редактирование столиков
* Управление зонами ресторана
* Автоматическое обновление статусов столиков

**Панель управления (Dashboard):**

* Просмотр сегодняшних бронирований
* Отображение доступных столиков
* Статистика загрузки в реальном времени
* Быстрые действия: "Бронь сейчас", "Бронь позже"
* Изменение статусов бронирований

**Интеллектуальные функции:**

* Поиск ближайшего доступного времени
* Автоматическая проверка пересечений бронирований
* Исключение столиков "На ремонте" из статистики

### 5.2. Технические требования:

* **Платформа:** Java 8+
* **Графический фреймворк:** JavaFX
* **База данных:** SQLite
* **Интервалы бронирования:** 15 минут
* **Время работы:** с 10:00 до 23:00
* **Статусы столиков:** Свободен, Занят, Забронирован, На ремонте

### 5.3. Основные модули:

* Главный экран с навигацией
* Панель управления (Dashboard)
* Модуль бронирований
* Модуль столиков
* Модуль зон

## 6\. Бизнес-правила

* Минимальный интервал брони: 15 минут
* Максимальная длительность: 180 минут
* Автоматическая проверка доступности столиков
* Столики на ремонте не участвуют в статистике
* Нельзя удалить столик с активными бронированиями

## 7\. Интерфейс пользователя

* Единый стиль навигации между разделами
* Табличное представление данных
* Диалоговые окна для добавления/редактирования
* Автоматическое обновление данных
